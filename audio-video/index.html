Audio & Video

Embedded Players

In general, I would recommend using an embedded player from a free audio or video hosting site whenever possible. These embedded players will automatically provide thumbnails and/or waveform preview, play/pause controls, progress bars, and handle media streaming/compression for you.

p5

If you absolutely insist on making your own player, or are simply looking for sound effects triggered by user actions, I would highly recommend going with p5. You can use the audio portion of p5 without using the actual canvas.

Audio in p5 requires a separate p5.sound library, which is available in the p5.js complete download (the p5.sound.min.js file that is part of this example folder).

The audio file will need to be included in your project folder. Ideally, it will be in the mp3 or aac format (to keep the file size small).

HTML Audio

You can also easily include an audio file using the audio HTML tag if all you need is a simple control bar. Again, make sure to keep file sizes small.

Video?

Honestly, I would just stay away from making your own video player for this project. Use YouTube (or Vimeo or whatever else you'd prefer that provides an embedded player) - otherwise, you have to start worrying about optimizing the video quality/codec, file size, etc.

If you ABSOLUTELY INSIST, there is a video HTML tag that works similarly to the audio tag (look it up on W3 Schools).
